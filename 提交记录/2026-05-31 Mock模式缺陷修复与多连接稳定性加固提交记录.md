## 一、背景（Background）

上一条提交 `44044ac` 完成了多服务器 MQTT 连接架构的完整实现，但在 Mock 模式联调中发现 11 项运行时缺陷：设备切换时数据残留、连接池回收不彻底、RPC 跨服务器互相干扰、操控栏 disabled 逻辑缺失、图表重复渲染等。

### 问题（Problem）

#### 1. MQTT 连接池对象生命周期缺陷

- `create()` 对同一 serverId 重复调用时未先销毁旧连接，造成连接泄漏
- `destroy()` 未清理 `_onlineClients` 缓存，导致已断开设备的在线状态残留
- `update()` 中 `ctx.deviceIds` 在 `destroy()` 后变为空 Set，重连后无法恢复设备订阅
- `switchFollowing()` 中 `clientAlive` 判断阻止了断开状态下取消关注旧设备主题

#### 2. RPC 跨服务器互相干扰

`clearPendingRpcs()` 无差别清空所有服务器的待处理 RPC。当某台服务器断线时，其他服务器上正常的 RPC 也被误杀。

#### 3. MQTT 消息解析缺乏异常防护

`state_changed`、`will`、`device_alarm`、`lowfreq` 主题直接 JSON.parse 无 try-catch，格式异常的消息会阻断整个消息路由。

#### 4. 设备切换时数据残留

store 清理在 pool 操作之后执行，时序上旧数据的清理发生在新设备数据已经到达之后，导致新旧数据短暂混合渲染。

#### 5. 设备删除时数据未清理

设置页删除设备后，若删除的是当前选中设备，waveform/data/alarm store 残留旧数据继续展示。

#### 6. UI 操控栏禁用的条件不完整

`StatusControlBar` 仅检查了 `selectedId`，未检查对应的 Device 对象是否存在。

#### 7. ServerSelect 布局溢出

添加设备弹窗中，Select 与按钮在同一行时超出容器边界，尤其是长文本选项导致右侧添加按钮被挤压不可见。

#### 8. 图表组件每次渲染都重建配置对象

`Cn2Chart` 和 `VisChart` 的数据派生（`samples.map`）和 ECharts `option` 对象在每次 render 时重新创建，产生大量不必要的 ECharts `setOption` 调用。

#### 9. Mock 连接 `sysInjectedRef` 模块级变量泄漏

`useMockGenerators` 的 `sysInjectedRef` 声明在模块顶层，多个组件实例共享同一 Set，跨组件互相污染。

#### 10. MockClient.end() 未清理订阅列表

`MockMqttClient.end()` 仅设 `connected = false`，未清空 `subscriptions`，重连后旧主题仍存在。

#### 11. waveform 二进制解析类型过窄

`parseWaveformBinary` 参数类型为 `ArrayBuffer`，但 `Uint8Array.buffer` 返回 `ArrayBufferLike`，需要不必要的类型断言。

## 二、解决方案（Solution）

### 整体思路

针对缺陷清单逐一修复：强化连接池生命周期管理、RPC 按 serverId 隔离清理、所有 JSON 解析路径增加 try-catch、数据清理前置、UI 条件补全、图表 useMemo 缓存。所有修复在小数据量范围（2-4 台服务器、数十台设备）内验证完毕。

### 具体实施

#### 1. 连接池生命周期加固 (`connectionPool.ts`)

- `create()` 入口增加重复检测：`if (this.servers.has(server.id)) this.destroy(server.id);`
- `destroy()` 补充 `this._onlineClients.delete(serverId);`
- `update()` 中 `const deviceIds = [...ctx.deviceIds]` 在 destroy 前保存，重建后恢复订阅
- `switchFollowing()` 移除 `clientAlive` 判断，直接执行 unsubscribe（连接断开时也应清理）

#### 2. RPC 按服务器隔离清理 (`rpc.ts`)

- `PendingRpc` 接口新增 `serverId: string` 字段
- `sendRpcCommand`/`sendRpcCommandLegacy` 在 `pendingRpcs.set()` 时传入 serverId
- `clearPendingRpcs(serverId?: string)` 改为按 serverId 过滤，reject 后从 Map 中删除对应项

#### 3. 消息解析全路径 try-catch (`router.ts`, `rpc.ts`)

- `router.ts` setupRouter / setupMqttRouter 中 6 处 JSON.parse 包裹 try-catch
- `rpc.ts` tryResolveRpc 中 payload 解析包裹 try-catch，失败时调用 `pending.reject()`
- 所有 catch 块输出日志后静默跳过（不中断路由）

#### 4. 设备切换数据清理前置 (`useMqttConnect.ts`)

```typescript
// 切换前先清空数据 stores
useCollectorStore.getState().reset();
useLaserStore.getState().reset();
useWaveformStore.getState().clear();
useDataStore.getState().clear();
useAlarmStore.getState().clear();

// 再执行连接/切换逻辑
if (prevDevice && prevDevice.serverId === device.serverId) { ... }
```

#### 5. 设备删除时级联清理 (`Settings/index.tsx`)

删除设备时调用 `pool.unsubscribeDevice()` 取消订阅；若为当前选中设备，额外清理 waveform/data/alarm store。

#### 6. UI 修复三项

- `StatusControlBar.tsx`：`allDisabled` 补上 `!selectedDevice` 条件
- `AddDeviceModal.module.css`：`.serverSelect` 增加 `min-width: 0; overflow: hidden;`，`.addBtn` 增加 `flex-shrink: 0`
- `AddDeviceModal.tsx`：给添加按钮绑定 className

#### 7. 图表 useMemo 优化 (`Cn2Chart.tsx`, `VisChart.tsx`)

timestamps/values 派生使用 `useMemo`，完整 option 对象使用 `useMemo`，避免每次 render 触发 ECharts 实例重建。

#### 8. 其他修复

- `useMockGenerators.ts`：`sysInjectedRef` 移入组件内 `useRef`
- `MockMqttClient.end()`：补充 `this.subscriptions = []`
- `binary.ts`：参数类型 `ArrayBuffer` → `ArrayBufferLike`
- `Sidebar.tsx`：`||` → `??`，空字符串视为有效连接状态

## 五、架构影响
