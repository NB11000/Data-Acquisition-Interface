## 一、背景（Background）

Cn2 折射率和 Vis 能见度两个低频图表存在多处问题：工具提示缺失、Mock 数据首点延迟、坐标轴刻度混乱、数据展示窗口不合理等。

### 问题（Problem）

#### 1. Vis 图表无 Tooltip

Vis 图表缺少 tooltip 配置，鼠标悬停时无法显示数据点的值，Cn2 图表正常。

#### 2. Mock 模式首次数据点延迟

`startMockLowFreq` 使用 7 秒间隔的 `setInterval`，不会立即执行第一次回调，导致采集开始后 Vis/Cn2 图表 7 秒空白。

#### 3. 数据点多了 X 轴标签拥挤

原始 `type: 'category'` 轴每个数据点一个标签，数据积累后标签互相重叠无法阅读。

#### 4. 坐标轴刻度对齐与数量不稳

经多轮方案迭代，时间轴刻度与数据点位置不对齐，刻度数量随字符串长度波动（2-3 个浮动）。

#### 5. X 轴标签显示冗余

ISO 时间戳标签（`2026-05-28T14:32:05`）过长，实际只需时间部分。

---

## 二、解决方案（Solution）

### 整体思路

统一两图表为 category 轴 + dataZoom 5 点窗口 + 程序化自动追尾 + HH:MM:SS 时间标签。Mock 生成器启动时立即 emit 首条数据。

### 具体实施

#### 1. Mock 生成器立即 emit 首帧

`mockWaveform.ts` / `mockLowFreq.ts` 将数据生成提取为 `tick` 函数，启动时先调用一次再交给 `setInterval`。

#### 2. 图表 5 点窗口 + 自动追尾

使用 `dataZoom: [{ type: 'inside' }]` 启用图表内交互缩放，`dispatchAction` 在每次新采样到达时将窗口定位到最后 5 个数据点。

#### 3. 坐标轴样式统一

- 取消 X/Y 轴线（`axisLine: { show: false }`）
- 取消 X 轴纵向网格线（`splitLine: { show: false }`）
- 保留 Y 轴横向网格线
- 取消边界间隙（`boundaryGap: false`）
- 强制显示全部标签（`axisLabel: { interval: 0 }`）

#### 4. X 轴标签简化为 HH:MM:SS

`utc.slice(11, 19)` 提取时间部分，5 个点恰好 5 个短标签，不挤不隐。
