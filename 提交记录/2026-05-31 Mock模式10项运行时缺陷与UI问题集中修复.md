## 一、背景（Background）

上版（44044ac）实现了多服务器 MQTT 连接架构的 8 个垂直切片，但 Mock 模式在实测中暴露出多个运行时 bug：RPC 超时、StrictMode 双监听器导致数据翻倍、Mock 功能遗漏、Mock/Real 类型污染、UI 状态丢失等。此外还有图表布局和断连重连相关的边角问题。

### 问题（Problem）

#### 1. Mock 模式 RPC 超时

`useRpcCommand` 使用已废弃的 `getMqttClient()` 旧单例（从未被 `initMqttClient` 初始化），RPC 请求无法到达 Mock 响应路由。

#### 2. StrictMode 双监听器导致低频数据翻倍

`setupRouter` 未返回清理函数，React StrictMode 双次挂载 effect 时 `pool.onMessage` 注册了 2 个监听器，每个消息触发 2 次 `dataStore.append()`。Vis/Cn² 图表出现重复数据点。

#### 3. Mock 模拟完整度不足

- `mockRpc.ts` 不推送 `state_changed` 事件（真实设备 RPC 后会推送），UI 依赖乐观更新绕过，换个组件就暴露
- Will/Alarm 消息完全不模拟

#### 4. Mock/Real 代码分离不彻底

- 新旧两套 `MockMqttClient` 并存（`src/mqtt/mockClient.ts` + `src/mock/mockMqttClient.ts`）
- `useMqttConnect.ts` 内嵌 3 处 `MQTT_MODE` 判断 + mock 生成器启动逻辑
- 5 处 `as unknown as MockClient` 类型断言绕过类型系统

#### 5. 图表与 UI 边角问题

- Dashboard wrapper 非 flex 容器 → ECharts 图表无限向下延申
- dataZoom 用 `useEffect` 异步设置 → 导航回页面时先闪烁全量数据再缩放
- Settings 本地 `useState serverStates` SPA 导航后丢失 → 连接状态始终显示"已断开"
- `ConnectionPool.destroy()` 清空回调后再 `end()` → "断开"按钮无反应

#### 6. 断连-重连后 RPC 超时

断开-重连创建新的 `MockMqttClient`，设备常驻主题（`$rpc`/`will`/`state_changed`/`alarm`）和跟随主题全丢失，RPC 注入无匹配订阅。

## 二、解决方案（Solution）

### 整体思路

修补 8 个垂直切片实现中的运行时缺陷：统一 `MqttClientLike` 接口消除类型污染、抽离 `useMockGenerators` 消除 Hook 污染、补全 mockRpc state_changed 注入、修复 StrictMode 和连接池生命周期导致的监听器/订阅泄漏。

### 具体实施

#### 1. Mock/Real 类型统一与接口扩展

`MqttClientLike` 接口新增 `injectMessage(topic, payload)` 方法，`RealMqttAdapter` 实现为 no-op。mock 工具函数（`mockRpc`/`mockWaveform`/`mockLowFreq`）统一接受 `MqttClientLike` 参数，消除所有 `as unknown as MockClient` 类型断言。

```typescript
// 变更前
import type { MockMqttClient } from './mockMqttClient';
export function handleMockRpc(mockClient: MockMqttClient, ...) { }
const client = pool.getClient(serverId) as unknown as MockClient;

// 变更后
import type { MqttClientLike } from '../mqtt/mqttClientLike';
export function handleMockRpc(mockClient: MqttClientLike, ...) { }
const client = pool.getClient(serverId); // 无需断言
```

#### 2. Mock 逻辑抽离（useMockGenerators）

新建 `src/hooks/useMockGenerators.ts`，将 $SYS 注入、波形/低频生成器启动逻辑从 `useMqttConnect.ts` 完全迁出。`useMqttConnect.ts` 零 `MQTT_MODE` 引用。

#### 3. Mock 功能补全

`mockRpc.ts` 在 RPC 成功后延迟 50ms 注入 `state_changed` 事件，模拟真实设备状态变更通知行为。Will/Alarm 模拟属更复杂需求，暂未实现（评估为低优先级）。

#### 4. StrictMode 修复

`setupRouter` 改为返回 `() => void` 清理函数，内部同时移除 `stateListener` 和 `messageListener`。`useMqttConnect` 捕获并在 effect 清理阶段调用。

#### 5. 图表与 UI 修复

- Dashboard wrapper 增加 `display: flex; flexDirection: column; overflow: hidden` 约束图表高度
- VisChart/Cn2Chart dataZoom 从 `useEffect` 改为 `useLayoutEffect`（同步绘制前设置）
- Settings 删除本地 `serverStates`，改为读 `serverStore.connectionState`（`useMqttConnect` 维护）
- `ConnectionPool.destroy()` 开头显式 `emitStateChange('disconnected')`

#### 6. 断连-重连订阅恢复

`ConnectionPool.create().onConnect` 中自动恢复该服务器下所有设备的常驻主题 + 当前选中设备的跟随主题，确保重连后 RPC/数据流完整。
