## 一、背景（Background）

MqttServer 数据模型中原有独立的布尔字段 `tls` 决定传输协议，brokerUrl 仅存储主机地址。这种分离设计导致 URL 拼接时出现重复前缀 bug，且存在数据冗余。同时 TLS 开关此前为自动跟随 brokerUrl 的只读控件，用户无法手动表达加密意图，且不支持 WebSocket 传输协议。

### 问题（Problem）

#### 1. URL 拼接重复前缀 bug

`connectionFactory` 中 `protocol = server.tls ? 'mqtts' : 'mqtt'` 与 `url = \`${protocol}://${server.brokerUrl}\`` 拼接，若用户 brokerUrl 已含协议前缀（如 `mqtts://host.com`），会拼出 `mqtts://mqtts://host.com:8883/mqtt`。

#### 2. tls 字段冗余

brokerUrl 本身即可承载协议信息，额外的 `tls: boolean` 是二义性来源，且与 brokerUrl 前缀可能冲突。

#### 3. TLS 开关无法手动控制

TLS 开关此前为 `disabled` 状态，自动跟随 brokerUrl 前缀切换。用户无法显式声明加密意图，操作路径不直观。

#### 4. 不支持 WebSocket 协议

brokerUrl 校验仅接受 `mqtt://` 和 `mqtts://` 前缀，不支持 mqtt.js 原生支持的 `ws://` 和 `wss://` 协议。

#### 5. AddDeviceModal 零服务器引导体验不佳

服务器列表为空时，AddDeviceModal 直接替换为 MqttServerModal，用户无法感知上下文切换。

#### 6. Mock 断连后注入记录未清理

`useMockGenerators` 在服务器断连/重连时未清理 `$SYS` 注入记录，重连后可能丢失注入。

#### 7. 端口字段交互问题

端口号使用 Select 下拉框，锁定为 1883/8883 两项，不支持自定义端口；且有默认值 8883 对非 TLS 场景产生误导。

---

## 二、解决方案（Solution）

### 整体思路

从数据模型层面删除 `tls: boolean` 字段，协议完全由 brokerUrl 前缀携带。TLS 开关改为用户手动控制的 UI-only 控件，保存时进行双向一致性校验。端口改为自由输入的 InputNumber，brokerUrl 前缀扩展支持 ws/wss。迁移逻辑自动为旧数据补全协议前缀。

### 具体实施

#### 1. 数据模型层删除 `tls` 字段

`src/stores/serverStore.ts` 和 `src/mqtt/types.ts` 的 `MqttServer` 接口删除 `tls: boolean`。`connectionFactory` 不再推导协议，URL 直接拼接 `${server.brokerUrl}:${server.port}`。`connectionPool.update()` 的 `needsReconnect` 从 5 项减至 4 项。`testConnection()` 签名删除 `tls` 参数。

#### 2. MqttServerModal TLS 开关手动化 + 协议扩展

- TLS Switch 变为手动控制，新建模式默认 OFF，编辑模式按 `caCert` 初始化
- 保存时双向校验：TLS ON → brokerUrl 须为 `mqtts://` 或 `wss://`；TLS OFF → 须为 `mqtt://` 或 `ws://`
- brokerUrl 正则扩展为 `/^(mqtts?|wss?):\/\/.+/`
- 端口从 Select 改为 InputNumber（1-65535 自由输入，无默认值）
- 删除 Form 的 `tls` Form.Item 及 `initialValues.tls`

#### 3. 数据迁移补全协议前缀

`runMigration()` 在生成 MqttServer 时，若旧 brokerUrl 无协议前缀，根据旧 `tls` 标志补 `mqtts://` 或 `mqtt://`。已有前缀则跳过，避免重复。

#### 4. AddDeviceModal 交互优化

- 移除零服务器时自动替换为 MqttServerModal 的行为
- 新增分区标题（"MQTT 服务器" / "设备信息"）
- 未选服务器时，Device ID、设备名称、添加按钮 disabled
- 服务器列表为空时下拉框显示"请先添加 MQTT 服务器"并 disabled

#### 5. Mock 断连清理注入记录

`useMockGenerators` 在服务器状态非 `connected` 时，遍历该服务器下所有设备清空 `sysInjectedRef`，确保重连后正确重新注入 `$SYS` 消息。

#### 6. Navbar 标签文案调整

导航栏 `/settings` 路由标签从"系统设置"改为"设备管理"，更准确反映当前页面功能。

## 五、架构影响

| 维度 | 变更前 | 变更后 |
|------|--------|--------|
| MqttServer 接口 | 含 `tls: boolean`、`isConnected` | 删除 `tls`，协议由 brokerUrl 前缀承载 |
| 连接 URL 构建 | `protocol + :// + brokerUrl + :port` | `${brokerUrl}:${port}` 直接透传 |

| 协议支持 | mqtt / mqtts | mqtt / mqtts / ws / wss |
| 端口输入 | Select 下拉（1883 / 8883），默认 8883 | InputNumber 自由输入，无默认值 |
| 数据迁移 | 直接搬运 brokerUrl | 根据旧 tls 补全 mqtts:// 或 mqtt:// 前缀 |
| testConnection 签名 | 5 参数（含 tls） | 4 参数 |
