## 一、背景（Background）

系统首次加载默认为浅色主题，用户每次打开页面都面临亮色界面。主题切换由旧版 Navbar 内联 Switch 直接操纵 DOM，无 localStorage 持久化——刷新即丢失偏好。CSS 变量存在两套命名前缀混用，`src/styles/` 和 `src/components/layout/` 目录冗余残留。

### 问题（Problem）

#### 1. 首次访问默认浅色主题

用户首次访问时看到浅色界面，需要手动切换才能进入暗色模式。无 localStorage 记录时默认为 `false`（浅色）。

#### 2. 主题偏好不持久化

旧版 Navbar 内联 Switch 直接调用 `document.body.classList.toggle`，未写入 localStorage。用户每次刷新页面都回到初始状态。

#### 3. CSS 变量命名不统一

实际生效的 `variables.css` 中混用无前缀（`--text-primary`）、不同前缀（`--statusbar-*`、`--chart-card-*`、`--navbar-text`）和 `--app-*` 前缀，增加维护成本。

#### 4. 冗余目录残留

`src/styles/`（55 行 variables.css + 29 行 global.css）和 `src/components/layout/`（AppLayout.tsx + Navbar.tsx + Sidebar.tsx）均未被任何文件引入。

#### 5. App.tsx 首帧闪烁风险

`isDark` 初始值通过 `document.body.classList.contains` 读取，而 ThemeSwitch 通过 `useEffect` 异步设置 body class，导致首帧渲染时可能状态不一致。

---

## 二、解决方案（Solution）

### 整体思路

统一主题系统：CSS 变量全量 `--app-*` 命名、ThemeSwitch 默认暗色并持久化到 localStorage、App.tsx 同步初始化消除闪烁、清理冗余目录。

### 具体实施

#### 1. ThemeSwitch 默认值改为暗色

`src/components/common/ThemeSwitch.tsx:11`：初始化逻辑从 `=== 'dark'` 改为 `!== 'light'`，无 localStorage 记录时天然默认为暗色。

```typescript
// 变更前
return localStorage.getItem(STORAGE_KEY) === 'dark';
// 变更后
return localStorage.getItem(STORAGE_KEY) !== 'light';
```

#### 2. App.tsx 与 ThemeSwitch 同源初始化

`src/App.tsx:12-14`：isDark 初始值从 body class 改为从 localStorage 读取，与 ThemeSwitch 共享 `app-theme` 键，消除首帧闪烁。

```typescript
// 变更前
const [isDark, setIsDark] = useState(
  () => document.body.classList.contains('dark-theme'),
);
// 变更后
const [isDark, setIsDark] = useState(
  () => localStorage.getItem('app-theme') !== 'light',
);
```

#### 3. CSS 变量统一为 27 个 `--app-*` 令牌

`src/assets/styles/variables.css`：从 15 变量扩展到 27 变量，全部 `--app-*` 前缀，`:root` 定义浅色、`body.dark-theme` 覆盖暗色。同时 `global.css` 补充滚动条样式和变量引用更新。

变量名映射表（10 对）：

| 旧名 | 新名 |
|------|------|
| `--app-navbar-bg` | `--app-nav-bg` |
| `--navbar-text` | `--app-nav-text` |
| `--text-primary` | `--app-text-primary` |
| `--text-secondary` | `--app-text-secondary` |
| `--statusbar-bg` | `--app-control-bar-bg` |
| `--statusbar-border` | `--app-control-bar-border` |
| `--chart-card-bg` | `--app-card-bg` |
| `--chart-card-border-top` | `--app-chart-card-border-top`（保留） |
| `--chart-card-padding` | `--app-chart-card-padding`（保留） |
| `--device-card-hover` | `--app-device-card-hover`（保留） |

#### 4. Navbar 整合 ThemeSwitch 组件

`src/components/Navbar.tsx`：移除内联 `<Switch>` + ☀️🌙 emoji 装饰 + `handleToggleDark` 回调，引入 `<ThemeSwitch />` 组件。保留 Tabs 导航、Logo、Avatar 不变。

#### 5. 13 个文件变量名批量迁移

8 个 CSS Module + 5 个 TSX 文件中的 31 处旧变量名全部更新为新 `--app-*` 命名：

- CSS Module：Navbar、Sidebar、DeviceCard、ChartCard、StatusControlBar、Settings、AddDeviceModal、antd-overrides
- 内联 style：Logs、History、Alerts、ErrorBoundary
- JS API：WaveformChart（`getResolvedColor`）

#### 6. 删除冗余目录

- `src/styles/`（variables.css + global.css，55+29 行）
- `src/components/layout/`（AppLayout + Navbar + Sidebar，38+29+92 行）
