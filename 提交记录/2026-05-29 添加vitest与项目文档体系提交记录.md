## 一、背景（Background）

项目进入规范化阶段，需要补充测试基础设施与项目文档体系。同时修复了自动发现设备弹窗中上线状态未对接真实 MQTT 连接状态的问题。

### 问题（Problem）

#### 1. 缺少单元测试框架

项目中无任何测试基础设施，缺少 `vitest` 测试运行器和相关脚本，无法编写和执行单元测试。

#### 2. 自动发现设备上线状态硬编码

`AutoDiscoverModal` 中添加设备时 `isOnline` 硬编码为 `false`，未使用 MQTT 连接返回的真实在线状态 (`m.online`)，导致设备上线状态不准确。

#### 3. 项目文档缺失

项目缺少系统性的设计文档、架构决策记录和问题跟踪体系，不利于团队协作和后续维护。

---

## 二、解决方案（Solution）

### 整体思路

添加 vitest 测试框架及配置，修复设备上线状态对接，建立完整的项目文档体系（PRD、ADR、实施计划、Issue 跟踪）。

### 具体实施

#### 1. 添加 vitest 测试框架

`package.json` 中新增 `vitest` 开发依赖及 `test` / `test:watch` 脚本。

```json
"test": "vitest run",
"test:watch": "vitest"
```

`vite.config.ts` 中加入 vitest 配置，限定测试文件范围为 `src/**/*.test.ts`：

```ts
/// <reference types="vitest/config" />
test: {
  include: ['src/**/*.test.ts'],
}
```

#### 2. 修复 AutoDiscoverModal isOnline

`src/components/modals/AutoDiscoverModal.tsx` 第 44 行，将 `isOnline` 从硬编码 `false` 改为使用 MQTT 连接返回的 `m.online` 字段。

```tsx
// 变更前
isOnline: false,
// 变更后
isOnline: m.online,
```

#### 3. 建立项目文档体系

新增以下文档目录结构：

```
docs/
├── PRD-001-*.md          # 设备 MQTT 配置重构
├── PRD-002-*.md          # MQTT 连接池与设备管理重构
├── IMPLEMENTATION-PLAN-001.md  # 实施计划
├── adr/
│   ├── 0001-per-device-mqtt-config.md
│   └── 0002-connection-pool.md
├── agents/
│   ├── domain.md
│   ├── issue-tracker.md
│   └── triage-labels.md
└── issues/
    ├── 001-009  # 9 个 Issue 文档
AGENTS.md          # Agent 上下文入口
CONTEXT.md         # 项目上下文说明
```
