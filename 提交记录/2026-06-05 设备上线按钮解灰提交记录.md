## 一、背景（Background）

设备从离线恢复上线后，右侧状态栏控制按钮一直置灰，必须手动刷新页面才能恢复。经诊断发现两个根因：

### 问题（Problem）

#### 1. system-state RPC 在设备离线时依然触发

点击离线设备选项卡时，`useMqttConnect` 的设备切换 Effect 不检查 `device.isOnline`，无条件调用 `sendStateRpcWithRetry`，导致反复弹出错误提示。

#### 2. 设备上线后按钮不自动解灰

设备上线（MQTT 重连）时 system-state RPC 返回 `processConnected: false`（采集子进程尚未启动），`allDisabled` 公式中的 `!processConnected` 导致全部按钮置灰。采集子进程就绪后推送 `state_changed` 事件（`processConnected: true`），但 `router.ts` handler 在 MQTT 回调中更新 Zustand store，此上下文中的 Zustand 订阅不触发 React 重渲染。

---

## 二、解决方案（Solution）

### 整体思路

1. **加 isOnline 门控** — 设备切换、MQTT 重连、StrictMode 兜底三条路径均检查 `device.isOnline`，离线设备跳过 RPC
2. **上线自动补发 RPC** — 新增 isOnline 监听 Effect，设备变上线时补发 system-state RPC
3. **桥接 MQTT 回调到 React** — 新建共享模块，`router.ts` state_changed handler 和 `handleResult` 更新 store 后触发桥接，强制 React 组件树重渲染
4. **移除 processConnected 门控** — `allDisabled` 不再包含 `!processConnected`，用户可点击按钮，后端 RPC 报错告知具体原因

### 具体实施

#### 1. isOnline 门控（三路径统一）

设备切换 Effect、onStateChange 重连路径、StrictMode 兜底路径均插入：

```typescript
if (!device.isOnline) return;
```

离线设备跳过 RPC，但 stores 清空和主题订阅切换照常执行。跳过 RPC 时仍需更新 `prevSelectedRef` 防止后续判断异常。

#### 2. 上线自动补发 Effect

监听当前选中设备的 `isOnline`，当值从 `false`/`null` 变为 `true` 时自动补发 system-state RPC。`prevOnlineRef` 去重，`selectedId` 变化时重置。

#### 3. React 桥接模块

`src/hooks/systemStateBridge.ts` — 模块级回调注册/触发，`useMqttConnect` mount 时注册（React setState），`router.ts` state_changed handler 和 `handleResult` 触发。双路径并行：直接 `applyState` + 桥接触发。

#### 4. allDisabled 简化

```diff
- const allDisabled = !mqttConnected || willReceived || !selectedId || !selectedDevice || !processConnected;
+ const allDisabled = !mqttConnected || willReceived || !selectedId || !selectedDevice;
```
