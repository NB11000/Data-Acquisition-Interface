分阶段实现计划

每个阶段以可运行、可验证的里程碑结束

Phase 1 — 项目骨架 + 布局

目标：Vite + React + TS 项目跑起来，布局框架可视

  • Vite 初始化项目，安装 antd / uPlot / mqtt 依赖
  • CSS 变量体系 (浅色/深色两套)
  • AppLayout (100vh flex)、Navbar (48px)、Sidebar (250px)、ContentArea
  • ThemeSwitch 可切换 dark-theme
  • ChartGrid 2×2 占位卡片 (写死假数据确认布局)
  • 验证：浏览器打开，深浅色切换正常，无外层滚动条

Phase 2 — 数据层

目标：纯 TS 模块可独立运行，不依赖 React

  • types/ — 所有 TS 类型定义 (SystemStateDto, CommandResult, MQTT 消息等)
  • mqtt/client.ts — MQTT 连接管理 (connect/disconnect/subscribe)
  • mqtt/rpc-manager.ts — RPC 请求/响应匹配 + 10s 超时
  • mqtt/topic-router.ts — 主题通配符路由
  • stores/ — waveform (RingBuffer), lowfreq, state, alarm
  • utils/ring-buffer.ts, binary-parser.ts
  • 验证：可在 Node/浏览器 console 中手动调用测试

Phase 3 — Hooks 适配层

目标：连接数据层和 UI 层

  • useMqttConnection — 设备连接/断连/自动重连
  • useSystemState — 订阅 state-store → React state (含 computeButtonDisabled)
  • useLowFreqData — 订阅 lowfreq → 单个图表组件 state
  • useWaveformData — 返回 RingBuffer ref，不触发 render
  • useRpcCommand — 发送 RPC + transition state + 乐观更新
  • useResizeObserver — 图表容器尺寸
  • 验证：hook 单元测试，mock store 验证状态流转

Phase 4 — 侧边栏 + 设备管理

目标：设备列表可交互，选中设备触发连接

  • DeviceCard (名称/ID/在线状态指示灯)
  • DeviceList (滚动容器，选中高亮)
  • AddDeviceButtons + AutoDiscoverModal + ManualAddModal
  • 选中设备 → useMqttConnection → 订阅该设备所有主题
  • 验证：Modal 表单交互正常，设备选中切换流畅

Phase 5 — 状态控制栏 + 强乐观交互

目标：按钮可操作，强乐观 UI 闭环

  • StatusControlBar + StatusIndicators + Clock
  • ControlButtons (采集组/激光组) + computeButtonDisabled 本地计算
  • RPC 命令发送 → 过渡态 → 乐观更新 → 纠偏
  • OfflineBanner (遗嘱触发 / MQTT 断连)
  • 验证：点击按钮 → 过渡态 → 终态 / 回退；10s 超时抖动

Phase 6 — 图表接入

目标：真实数据驱动图表渲染

  • WaveformChart — uPlot 双Y轴，命令式更新 RingBuffer
  • VisChart + Cn2Chart — uPlot 时间序列折线
  • MultiParamCard — Empty 占位 (六要素暂不可用)
  • ResizeObserver 自适应高度
  • 设备切换时清理上一设备数据
  • 验证：100ms 波形流畅滚动，无卡顿；7s 低频正常更新

依赖关系

  Phase 1 (骨架) ──┐
                   ├──▶ Phase 4 (侧边栏) ──▶ Phase 5 (控制栏) ──▶ Phase 6 (图表)
  Phase 2 (数据层)─┤
                   │
  Phase 3 (hooks) ─┘

Phase 2 和 Phase 3 是基础设施，完成后 Phase 4/5/6 可以相对独立推进。
