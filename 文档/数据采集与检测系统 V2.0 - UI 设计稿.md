# 2. 核心区块详细设计

## 2.1 顶部导航栏 (Navbar)

高度：固定 48px。

左侧：系统 Logo + "数据采集与检测系统 V2.0"。

中侧（Menu）：采用深色/浅色主题适配的水平 Tabs。包含：数据中心 (高亮)、告警中心、历史数据、系统设置、日志查看。

右侧：主题切换 Switch (☀️/🌙 切换 dark-theme)，用户头像/退出登录。

## 2.2 侧边栏与设备管理 (Sidebar)

布局：宽度 250px，顶部是设备总数统计和搜索框，中部为滚动设备列表，底部为固定的操作按钮。

### 设备卡片 (Device Card)：

- 标题：用户自定义设备名称（如 "一号高塔节点"）。

- 副标题：MachineId（如 daq-srv-01，用灰色小字显示，作为唯一标识）。

- 状态指示灯：基于 MQTT 连接状态显示（🟢 绿点=已连接，🔴 红点=已离线/遗嘱状态）。

交互：点击卡片高亮选中，右侧工作区立即切换该设备的 MQTT 数据流。

### 底部固定区：

水平排列两个主按钮：[ 🔍 自动发现 ] 和 [ ➕ 手动添加 ]。按钮始终吸底，不随设备列表滚动。

## 2.3 状态控制栏 (StatusControlBar)

此区域为本次重构核心，由原底部面板上移，承载“强乐观交互”逻辑。

视觉容器：采用带有细边框和极浅背景色（暗色模式下为深灰半透明）的 Card 样式，保留原 MainPage.css 中的渐变和阴影效果。

### 第一行（状态栏）：

- 左侧：当前选中设备名称与 MachineId。

- 中部：指示灯组件。MQTT 状态: 🟢已连接 | 采集状态: 🟢采集中 / 🔴未采集 | 激光状态: 🟡已连接。

- 右侧：当前系统时间（数字时钟动态跳动）。

### 第二行（操作组）：

- 采集控制组：[ 打开采集卡 ] [ 🔴 开始采集 / 停止采集 ]

- 激光控制组：[ 连接激光 ] [ ☢️ 开启激光 / 关闭激光 ]

强乐观 UI 反馈：当用户点击【开始采集】，按钮立即变灰并显示“发送中...”。收到 MQTT RPC 成功响应后，瞬间变为绿色“采集中”状态，无需等待硬件回传，提供极致的点击跟手感。如果遇到硬件异常（如 Broker 发送遗嘱），所有按钮强制置灰不可点。

## 2.4 核心图表区 (2x2 Grid)

严格遵守《重构设计文档第12节》的视觉硬约束，不改变原有图表视觉风格。

网格布局：使用 Ant Design 的 Row 和 Col，设置 gutter={[16, 16]}，实现绝对均分的 2x2 网格。

高度自适应：容器使用 flex: 1 填满剩余屏幕，每个 Chart Container 内部通过 ResizeObserver 计算 uPlot 的画布高度，确保在 100vh 限制下不出现外层滚动条，图表完美自适应。

### 图表分布：

- 左上：双通道电压波形图（高速刷新，订阅 QoS 0）。

- 右上：Vis 能见度（时间序列折线图，订阅 QoS 1）。

- 左下：多参数六要素（由于后端暂无数据，此处显示标准 Card 样式，中央展示友好的 Empty 状态：“六要素数据暂不可用”，绝不破坏 2x2 结构）。

- 右下：Cn² 折射率（时间序列折线图，订阅 QoS 1）。

# 3. 核心弹窗交互设计 (Modals)

## 3.1 自动发现弹窗 (AutoDiscoverModal)

旨在通过扫描 MQTT Broker 自动带出设备。

### 第一步（连接 Broker）：

- 表单：Broker 地址、端口 (8883)、用户名、密码。

- 操作：点击 [ 开始扫描 ]，按钮显示 Loading 动画。

### 第二步（扫描结果呈现）：

界面滑出表格列表，监听 daq/+/events/state_changed 抓取保留消息。

表格列：选中框 | 发现的 MachineId | 当前状态 (在线/离线)。

### 第三步（确认添加）：

点击 [ 添加选中设备 ]。

## 3.2 手动添加弹窗 (ManualAddModal)

标准表单交互。

表单项：设备名称 (输入框), Broker 地址 (输入框), 端口号 (数字), MachineId (关键标识，必填), 认证信息 (用户名/密码), 开启 TLS (Switch 开关)。

操作：点击 [ 测试连接并保存 ]。系统将尝试建立独立 MQTT 连接，成功后直接将设备写入侧边栏列表。

# 4. 全局状态与异常提示设计

由于引入了 MQTT 和离线遗嘱机制，系统需要更显著的异常感知 UI：

## MQTT 断连 / 遗嘱触发 (Will Message)：

触发条件：收到 daq/{machineId}/events/will。

### UI 表现：

- 状态控制栏下方立即滑出一个 全屏宽度的红色横幅 (Banner)：“⚠️ 设备 [daq-srv-01] 已离线，主控进程可能已崩溃或网络断开”。

- 状态栏的四个控制按钮全部强制附带 disabled 样式和灰色蒙层。

- 图表区冻结渲染，保留最后一帧数据。

## RPC 响应超时 (10s)：

UI 表现：如果操作按钮（如“开启激光”）转圈超过 10 秒，按钮抖动并退回原始状态，通过全局 Message.error('服务端无响应 (RPC超时)') 在右上角进行轻提示。

# 5. CSS 变量与主题迁移说明

为了确保“改版不改皮”，本设计要求前端开发在重构时遵守以下 CSS 规范：

- 保留 Token：从旧版 MainPage.css 抽离的色值必须挂载到新布局容器上。例如背景色继续使用 var(--app-bg-color)。

- 图表卡片：图表外部包裹的 Card 必须保留现有的 borderTop 颜色标识条，卡片内边距严格维持 padding: 8px。

- 深浅色切换：Navbar 上的主题切换按钮将继续触发 document.body.classList.toggle('dark-theme')，所有新加的组件（StatusControlBar、Modals）必须同时实现浅色背景（如 #ffffff）和深色背景（如 #141414）的两套配置。
